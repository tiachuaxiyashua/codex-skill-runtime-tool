# Proposal: Runtime Architecture Facts Check

## Why

`项目问题分析图谱.md` 已经能指出当前仓库的几个真实结构问题，但它仍然是一次性手工写作，缺少可复核的事实来源。随着仓库持续变化，文档很容易再次漂移，变成“看起来像分析、但不能复算”的静态说明。

## What Changes

- 增加一个轻量的仓库架构事实校验脚本，按当前工作区动态统计：
  - 关键文件行数
  - tracked diff 规模
  - skill 包顶层骨架
  - Python 文件总量
- 把 `项目问题分析图谱.md` 的口径收敛为“当前快照 + 可再生成”。
- 为事实校验脚本增加自检模式，确保统计和分类逻辑本身可运行。

## Scope

这次 change 只做“图谱事实的可再生成”与“文档口径收敛”，不拆分 `server.py`、`tool_executor.py` 或 `selftest.py`。

## Non-Goals

- 不重构 runtime 核心执行流。
- 不更改 UI 交互。
- 不引入新的领域能力或插件机制。

## Acceptance

- 事实校验脚本可以在仓库根目录运行并输出当前快照。
- 自检模式能通过。
- `项目问题分析图谱.md` 能明确说明这些数字是可生成的快照，而不是永久真值。
- OpenSpec 能通过校验。
