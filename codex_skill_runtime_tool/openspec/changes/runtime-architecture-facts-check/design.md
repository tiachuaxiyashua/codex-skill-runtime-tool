# Design: Runtime Architecture Facts Check

## Data Source

The script reads only local repository state:

- file system for file existence and line counts
- `git diff --stat` / `git status --short` for workspace scale
- top-level skill directories for structure comparison

## Output

The script emits a small JSON object with:

- repository root
- current file counts
- selected file line counts
- tracked diff stats
- top-level skill skeletons

It can also render a short markdown summary for manual review.

## Selftest

The selftest mode checks:

- the repository root resolves correctly
- line-count collection returns integers
- tracked diff parsing returns the expected shape
- `art_pipeline_skill` and `audio_pipeline_skill` have the same large-skill skeleton class
- `godot_tool_bridge_skill` is correctly recognized as the smaller bridge-style skeleton

## Documentation Contract

`项目问题分析图谱.md` should keep the human-readable diagnosis, but the footer must say the numbers are a snapshot that can be regenerated by the audit script.
